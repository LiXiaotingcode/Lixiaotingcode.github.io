Linux 学习干货整理：从入门到能独立排查问题

发表于2026-05-10|更新于2026-05-10|Linux学习
|浏览量:

Linux 不是“会几个命令”就算学会了。真正有用的学习方式，是把它当成一个完整系统来理解：文件怎么组织、权限怎么控制、进程怎么运行、服务怎么启动、网络怎么通信、问题怎么排查。

这篇文章按“能真正上手”的顺序整理，适合做一篇长期保留的学习笔记。

1. 先理解 Linux 到底是什么

Linux 严格来说是内核，我们平时说的 Ubuntu、CentOS、Debian、Arch 等，其实是基于 Linux 内核的发行版。

一个完整的 Linux 系统通常包含：

  • 内核：负责进程调度、内存管理、硬件驱动、文件系统
  • Shell：命令行解释器，最常见的是 bash
  • GNU 工具链：比如 ls、cp、grep、find、tar
  • 包管理器：比如 apt、yum、dnf
  • 服务管理工具：现代系统常见的是 systemd

2. 文件系统是最基础的认知

Linux 中“一切皆文件”这句话非常重要。普通文件、目录、设备、管道、套接字，本质上都以文件接口来组织。

常见目录作用：

  • /：根目录
  • /home：普通用户家目录
  • /root：root 用户家目录
  • /etc：系统配置文件
  • /var：日志、缓存等经常变化的数据
  • /usr：用户级程序和共享资源
  • /tmp：临时文件
  • /proc：内核和进程信息的虚拟文件系统

常用命令：

pwd
ls -la
cd /etc
mkdir demo
touch a.txt
cp a.txt b.txt
mv b.txt notes.txt
rm notes.txt
rm -r demo

3. 学会查看文件内容

很多 Linux 工作都围绕“看配置、看日志、看输出”展开。

cat /etc/hostname
less /var/log/syslog
head -n 20 /etc/passwd
tail -n 20 /var/log/syslog
tail -f /var/log/nginx/access.log
grep -n "listen" /etc/nginx/nginx.conf
grep -r "server" /etc/nginx

建议尽快熟悉：

  • less：看大文件
  • tail -f：盯日志
  • grep：查文本

4. 权限模型必须吃透

示例：

-rw-r--r-- 1 user user 1200 May 10 10:00 test.txt

权限字符含义：

  • r：读
  • w：写
  • x：执行

常见权限命令：

chmod u+x run.sh
chmod 644 test.txt
chmod 755 script.sh
chown user:user test.txt

很多“文件明明存在却访问不了”的问题，根源都在目录权限。

5. 用户、用户组与 sudo

Linux 是多用户系统，所以身份很重要。

whoami
id
groups
sudo command
su -

日常管理优先用 sudo，不要长期直接以 root 身份操作。

6. 进程管理是日常高频操作

ps aux
ps aux | grep nginx
top
pgrep node
kill 1234
pkill nginx
nohup python app.py > app.log 2>&1 &

建议先用普通 kill，kill -9 留作最后手段。

7. 软件安装要分发行版理解

Debian / Ubuntu：

sudo apt update
sudo apt install nginx
sudo apt remove nginx

CentOS / Rocky / RHEL：

sudo yum install nginx
sudo dnf install nginx

不要随便复制网上命令，先确认你的发行版和包管理器。

8. systemd 和服务管理

sudo systemctl start nginx
sudo systemctl stop nginx
sudo systemctl restart nginx
sudo systemctl status nginx
sudo systemctl enable nginx
journalctl -u nginx -n 50

一个服务启动失败时，先看 systemctl status，再看 journalctl。

9. 网络命令必须会一套

ip addr
ip route
ping 8.8.8.8
curl http://127.0.0.1:3000
curl -I https://example.com
ss -tulpn
nslookup example.com

实际排障思路：

  • 能不能通 IP
  • 能不能解析域名
  • 服务有没有监听端口
  • 防火墙有没有放行

10. 日志是排错第一现场

常见日志目录：

  • /var/log/syslog
  • /var/log/messages
  • /var/log/auth.log
  • /var/log/nginx/access.log
  • /var/log/nginx/error.log

高效看日志：

tail -f /var/log/nginx/error.log
grep -i error /var/log/syslog
grep -n "failed" /var/log/auth.log

11. 压缩、归档与传输

tar -czf backup.tar.gz /home/user/data
tar -xzf backup.tar.gz
zip -r site.zip public
unzip site.zip
scp file.txt user@server:/home/user/
rsync -av project/ user@server:/home/user/project/

12. Vim 不一定精通，但至少要能用

最常用的几个：

  • i：进入插入模式
  • Esc：退出插入模式
  • :w：保存
  • :q：退出
  • :wq：保存并退出
  • :q!：不保存强制退出

13. Shell 管道与重定向要熟练

ps aux | grep nginx
echo "hello" > a.txt
echo "world" >> a.txt
command > out.log
command 2> err.log
command > all.log 2>&1

重点理解：

  • |：把前一个命令输出交给后一个命令处理
  • >：覆盖写入
  • >>：追加写入
  • 2>：重定向错误输出

14. 学 Linux 时最实用的排障方法

通用排查流程：

  1. 先看报错原文
  2. 确认是权限、路径、端口还是配置问题
  3. 看日志
  4. 看进程
  5. 看端口
  6. 看配置
  7. 看资源

网页打不开时，可以先查：

systemctl status nginx
ss -tulpn | grep 80
curl http://127.0.0.1
tail -n 50 /var/log/nginx/error.log

15. Linux 学习路线建议

建议按这个顺序学：

  1. 文件和目录操作
  2. 文件查看与搜索
  3. 权限和用户
  4. 进程管理
  5. 软件安装
  6. 服务管理
  7. 网络排查
  8. 日志分析
  9. Shell 脚本入门

16. 总结

Linux 真正有价值的地方，不是让你会敲很多命令，而是让你具备一种稳定的系统化排查能力。

当你面对“服务起不来、网站打不开、权限不够、日志报错、端口冲突”这些实际问题时，能沿着正确路径一步步查下去，这才算真正入门。
